1.Maintain a data analytics solution (25–30%)
1.1. Implement security and governance
- 1.1.1. Implement workspace-level access controls
- 1.1.2. Implement item-level access controls
- 1.1.3. Implement row-level, column-level, object-level, and file-level access control
- 1.1.4. Apply sensitivity labels to items
- 1.1.5. Endorse items
2.1. Maintain the analytics development lifecycle
- 1.2.1. Configure version control for a workspace
- 1.2.2. Create and manage a Power BI Desktop project (.pbip)
- 1.2.3. Create and configure deployment pipelines
- 1.2.4. Perform impact analysis of downstream dependencies from lakehouses, data warehouses, dataflows, and semantic models
- 1.2.5. Deploy and manage semantic models by using the XMLA endpoint
- 1.2.6. Create and update reusable assets, including Power BI template (.pbit) files, Power BI data source (.pbids) files, and shared semantic models
2.Prepare data (45–50%)
2.1. Get data
- 2.1.1. Create a data connection
- 2.1.2. Discover data by using OneLake data hub and real-time hub
- 2.1.3. Ingest or access data as needed
- 2.1.4. Choose between a lakehouse, warehouse, or eventhouse
- 2.1.5. Implement OneLake integration for eventhouse and semantic models
2.2. Transform data
- 2.2.1. Create views, functions, and stored procedures
- 2.2.2. Enrich data by adding new columns or tables
- 2.2.3. Implement a star schema for a lakehouse or warehouse
- 2.2.4. Denormalize data
- 2.2.5. Aggregate data
- 2.2.6. Merge or join data
- 2.2.7. Identify and resolve duplicate data, missing data, or null values
- 2.2.8. Convert column data types
- 2.2.9. Filter data
2.3. Query and analyze data
- 2.3.1. Select, filter, and aggregate data by using the Visual Query Editor
- 2.3.2. Select, filter, and aggregate data by using SQL
- 2.3.3. Select, filter, and aggregate data by using KQL
3.Implement and manage semantic models (25–30%)
3.1. Design and build semantic models
- 3.1.1. Choose a storage mode
- 3.1.2. Implement a star schema for a semantic model
- 3.1.3. Implement relationships, such as bridge tables and many-to-many relationships
- 3.1.4. Write calculations that use DAX variables and functions, such as iterators, table filtering, windowing, and information functions
- 3.1.5. Implement calculation groups, dynamic format strings, and field parameters
- 3.1.6. Identify use cases for and configure large semantic model storage format
- 3.1.7. Design and build composite models
3.2. Optimize enterprise-scale semantic models
- 3.2.1. Implement performance improvements in queries and report visuals
- 3.2.2. Improve DAX performance
- 3.2.3. Configure Direct Lake, including default fallback and refresh behavior
- 3.2.4. Implement incremental refresh for semantic models
